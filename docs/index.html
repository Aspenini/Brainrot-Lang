🧠💀 Brainrot Lang (BRL)

Meme BASIC with emojis — usable, cursed, and cross-platform.

Overview

Brainrot Lang (BRL) is a meme-first scripting language with readable keywords (RIZZ, SKIBIDI) and emoji operators (💀, 😭, 🔥…). It’s Turing-complete, has dev-mode sanity, and abstracts OS/graphics behind a tiny HAL so your code stays portable and self-hostable.

  • File extension: .brainrot
  • Required program fence:
💀💀💀
... your code ...
💀💀💀

Encoding: UTF-8 (emojis are first-class Unicode).

Modes & Rituals

  • NO CAP — run top→bottom (otherwise executes bottom→top).
  • DEV ENERGY — disables gaslighting, disables variable decay, keeps outputs normal, forward exec.

Comments: // line, /* block */

Brainrot Data Types

BRL supports exactly 4 core brainrot types:

TypeDescriptionExample
CHONKInteger (64-bit)420, 69
TWIGFloat (double precision)3.14, 2.718
BLABBERString (UTF-8)"SKIBIDI TOILET"
HOARDList/Array (dynamic)[1, 2, "GYATT"]

Type coercion: Automatic conversion between CHONK and TWIG as needed. Strings can be converted with STRINGIFY.

Braincells (Variables)

There are exactly 7 global braincells: sigma, gyatt, skull, npc, goon, fanum, aura.

  • Declare/assign: FANUMTAX name = expr
  • Direct reassign: name = expr
  • Decay: each cell's value can be used 3 times, then becomes "cooked" (unset). Disabled in DEV ENERGY.
  • Scope: All braincells are global. Functions create temporary local copies that shadow globals during execution.
FANUMTAX sigma = 3
FANUMTAX aura = sigma 💀 4
sigma = sigma 😭 1

Comparisons & Membership

  • IS — equal
  • COOKED — not equal
  • GYATTIER — greater than
  • NPC — less than
  • IN — membership (in list/string)

Blocks (no braces)

  • Begin block: LOCK IN
  • End block: IT’S OVER (also accept ITS OVER)

I/O

  • GYATT expr — print (every 3rd print becomes aLiTeRnAtInG unless DEV ENERGY).
  • NPC — input (user must type skibidi toilet then real input). Input lands in braincell aura.

RNG — the Holy 67

67 FROM a TO b
  • Returns random int in [a, b].
  • Edge case: 67 FROM 67 TO 67 prints “THE HOLY NUMBER 💀” and halts.

Time

  • CLOCKED — current time in seconds (float) → stored in sigma.
  • SNOOZE ms N — sleep for N milliseconds.

Memory Management & Cleanup

  • Automatic cleanup: All braincells auto-clean when program ends or variables "cook" (3-use limit)
  • YEET buf — manually free buffer/pixel memory
  • CLEANUP — force garbage collection (dev tool)
  • Memory leaks: Disabled in DEV ENERGY — automatic reference counting
  • Stack overflow: Infinite recursion protection (max 1000 calls)
FANUMTAX buf = SPAWN BYTES 1000
// ... use buffer ...
YEET buf  // manual cleanup
// or let it auto-clean when program ends

Buffers & Pixels

Typed Byte Buffers

  • SPAWN BYTES n → new byte buffer (handle in goon).
  • SCOOP BYTE buf AT i → read byte (to aura).
  • PLUG BYTE buf AT i WITH v → write byte.
  • SQUADLEN buf → length (int) → to sigma.

Framebuffers & Draw

  • OPEN RIZZPORT w h TITLE "txt" — open the render surface (backend abstracted).
  • SPAWN PIXELS w h — offscreen framebuffer handle → stored where you assign it.
  • DROP PIXEL x y rgba — set pixel on the active surface.
  • PUT PX fb X x Y y COLOR c — set pixel on offscreen fb.
  • GET PX fb X x Y y — get color → to aura.
  • BLIT PIXELS fb — blit offscreen fb onto RIZZPORT.
  • WIPE rgba — clear screen.
  • SHOWTIME — present/swap buffers.
  • LOCKOFF — close the window/surface.

Color Helpers

  • PACK RGBA r g b a → returns 0xRRGGBBAA → to aura.
  • UNPACK RGBA color → returns record DRIP r IS .., g IS .., b IS .., a IS .. ENDDRIP → to aura.

2D Indexing Helpers

  • AT2D base WIDTH w X x Y y → 1D index (for grids) → to aura.

Portable Image IO (PPM)

  • WRITE PPM fb TO "out.ppm" — dump framebuffer to PPM file.
  • READ PPM "tex.ppm" — load PPM to a pixel buffer (handle) or error.

Input (Keyboard)

  • KEYS DOWN — list of keys currently pressed → to fanum.
  • Alias: TOILET CHECK does the same.

Literals & Escape Sequences

String Literals (BLABBER)

  • Delimiters: Double quotes "text" or single quotes 'text'
  • Escape sequences:
    • \\ — backslash
    • \" — double quote
    • \' — single quote
    • \n — newline
    • \t — tab
    • \r — carriage return
    • \u{1F480} — Unicode escape (💀)

Number Literals

  • Integers (CHONK): 420, 0x69 (hex), 0o77 (octal), 0b1010 (binary)
  • Floats (TWIG): 3.14, 2.718e0 (scientific), 1e-3
  • Special values: INF (infinity), NAN (not a number)

List Literals (HOARD)

  • Syntax: [item1, item2, item3]
  • Mixed types: [1, "GYATT", 3.14]
  • Nested: [[1, 2], [3, 4]]

Strings & Basic Ops

  • MIX a WITH b — concat → to aura.
  • STRINGIFY x — to string → to aura.
  • LEN x — length → to sigma.
  • SPLIT text BY "sep" — list → to fanum.
  • JOIN list BY "sep" — string → to aura.
  • SLICE text FROM i TO j — substring → to aura.

Paths & Dirs (Portable)

  • GYATTPATH a b c — join path segments.
  • WHERE AM I — current working dir → to aura.
  • TP "path" — change dir.
  • RIZZHOME — home dir → to aura.
  • SKIBIDITEMP — temp dir → to aura.

Files & FS

  • READDROP "file.txt" — read text file → W/L packet.
  • SPITFILE "file.txt" WITH text — write/overwrite.
  • APPENDFILE "file.txt" WITH text — append.
  • LISTUP "dir" — list entries → to fanum.
  • MAKEHOUSE "dir" — mkdir.
  • NUKEFILE "path" — remove file.
  • NUKEHOUSE "dir" — remove directory.
  • STATS "path" — returns DRIP size IS n, isdir IS true/false, mtime IS secs ENDDRIP → to aura.

Concurrency & Async

  • SPIDERSPAWN func USING args LOCK IN ... IT'S OVER — spawn concurrent task
  • LOITER task — wait for task completion → W/L packet
  • CHILL ms N — non-blocking sleep (yield to other tasks)
  • Threading model: Cooperative multitasking with yield points
  • Shared state: All 7 braincells are thread-safe with atomic operations
  • Deadlock protection: Automatic timeout after 5 seconds
FANUMTAX task = SPIDERSPAWN slow_function USING 42 LOCK IN
  // ... do other work ...
  FANUMTAX result = LOITER task
  RIZZ IS W result LOCK IN
    GYATT "Task completed!"

Processes & Env

  • GYATTARGS — CLI args → to fanum.
  • GYATTENV "NAME" — env var → W/L packet.
  • DROP NPC "prog" WITH SQUAD "a","b" ENDSQUAD CATCH — safe spawn (no shell) → W packet: DRIP code, out, err ENDDRIP.
  • DROP SHELL "echo SKIBIDI" CATCH — run via shell (portable wrapper) → W packet.
  • DIPSET n — exit program with status code n.

Platform & Capabilities

  • PLATFORMGYATT — returns "windows" | "linux" | "macos" → to aura.
  • CANCOOK "feature" — true/false for features like "shell", "subprocess", "binary-io" → to sigma (1 or 0).

Packets (Portable Error Model)

Risky ops return W/L packets. Helpers:

  • IS W x, IS L x — type check
  • TAKE W x, TAKE L x — extract value / error
FANUMTAX r = READDROP "data.txt"
RIZZ IS W r LOCK IN
  FANUMTAX txt = TAKE W r
  GYATT txt
NO RIZZ LOCK IN
  GYATT "YOU FELL OFF"
IT’S OVER

Debugging & Error Reporting

  • SNIFF expr — print debug info (variable name, value, type)
  • STALK "message" — debug trace with timestamp
  • FREEZEFRAME — pause execution and show stack trace
  • BRAINLEAK — print current call stack
  • MINDPALACE — show braincell states and memory usage
  • Error levels: LOWKEY (warning), MID (error), COOKED (fatal)
  • Debug mode: DEBUG MODE — enables verbose error messages instead of "YOU FELL OFF"
DEBUG MODE
FANUMTAX sigma = 42
SNIFF sigma  // prints: sigma = 42 (CHONK)
STALK "About to do math"
FANUMTAX result = sigma 💀 1

Execution Quirks

  • Backwards execution unless you say NO CAP.
  • Gaslighting: sometimes prints RIZZLESS. (Off in DEV ENERGY.)
  • Errors: always render as YOU FELL OFF (unless in DEBUG MODE).
  • Program ends at closing 💀💀💀, or via DIPSET n, or the holy 67 edge case.

Lexing & Aliases

  • Emoji ops accept a single alias (table above). Aliases are case-insensitive.
  • Multiword tokens are single keywords (e.g., LOCK IN, NO RIZZ, OPEN RIZZPORT, SPAWN BYTES).
  • Unicode normalized (NFKC) before tokenizing.

Performance & Optimization

  • Interpreter overhead: ~10-50x slower than native code (typical for interpreted languages)
  • HYPERSKOOM — enables JIT compilation for hot loops (experimental)
  • SPEEDRUN "name" — measure execution time of code block
  • Memory usage: ~1MB base + 8 bytes per braincell + buffer allocations
  • Optimization tips:
    • Use DEV ENERGY to disable variable decay overhead
    • Minimize string concatenation — use JOIN for multiple strings
    • Reuse buffers instead of creating new ones
    • Prefer integer math over float when possible
  • Profiling: PROFILE ON / PROFILE OFF — generates performance report
PROFILE ON
FANUMTAX start = CLOCKED
// ... your code ...
FANUMTAX end = CLOCKED
GYATT "Execution time: " 💀 (end 😭 start) 💀 " seconds"
PROFILE OFF

Portability & Backends

BRL code only sees the meme APIs above. The runtime targets a tiny "Rizzport HAL" behind the scenes (headless/PPM, terminal, native, web, etc.). No SDL/Tk in userland; self-hosting path stays clean.
